## Nits (non-blocking)

1. **The third guard (`Number.isFinite` catch-all) is correctly implemented but has no dedicated regression test.** Confirmed by mutation testing: deleting `Number.isFinite(bpm) ? bpm : undefined` (replacing it with a bare `return bpm;`) does not fail any of the 6 current tests — every fixture either trips guard (a) or (b) first, or lands on a healthy finite result. A concrete case this guard alone protects against: a marker with `sample_time: NaN` (plausible as malformed/corrupt Ableton warp data). `NaN <= 0` evaluates to `false` in JS, so guard (b)'s `sampleTimeSpan <= 0` check does **not** catch it — the value falls through to `bpm = NaN`, caught only by guard (c). I verified this with a throwaway script. Worth noting beyond "just a coverage gap": if guard (c) were ever lost, the call site's `if (bpm === undefined)` check would not fire for a `NaN` result (`NaN === undefined` is `false`), so the clip-attributed warning would silently not log — the operator would only see `setTempo`'s generic `Ignoring setTempo(NaN): not a finite number` message with no clip name, since `bpm !== undefined` would still be `true` and `setTempo(NaN)` would still be called (safely rejected there, but with less diagnostic context). Not required — the ticket's explicit minimum test list (0, 1, 2-same-time, healthy) doesn't demand this fixture, and the guard is correct today — but a `{ beat_time: NaN, sample_time: 5 }`-style fixture would close a real regression-lock gap cheaply.
2. **Cosmetic: the single-marker test's inline comment is slightly imprecise about which guard fires.** `backend/adapter/test/AbletonAdapter.test.ts:87` labels the case "(zero sample-time span)", but for a one-element array the function actually short-circuits on guard (a) (`length < 2`) before any span is ever computed — it never reaches the span check. The assertion and coverage are still correct and meaningful (a single marker is a real degenerate case regardless of which guard catches it); this is purely a comment-wording nit, not a functional or coverage issue.
